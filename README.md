# gAcss (gAccess) Audit Tool

**gAcss** — это консольный инструмент для проведения аудита безопасности и анализа структуры файлов в Google Drive. Приложение позволяет быстро просканировать содержимое облачного хранилища, отфильтровать права доступа и выгрузить подробные отчеты в формате Excel.

---

## Основные возможности

* **Полный сканирование:** получение данных о всех файлах на диске.
* **Рекурсивный аудит папок:** анализ содержимого конкретных папок по ссылке или ID.
* **Фильтрация по участникам:** поиск файлов, к которым имеет доступ конкретный пользователь (по Email).
* **Экспорт в XLSX:** формирование подробных таблиц с именами, ссылками, ролями доступа, владельцами и размерами файлов.
* **Безопасный выход:** возможность мгновенного сброса авторизации и смены аккаунта.

## Инструкция по получению credentials.json

Чтобы программа **gAcss** могла безопасно получить доступ к вашему Google Drive, необходимо создать персональный «ключ доступа» в консоли Google Cloud. Это бесплатно и занимает около 5 минут.

---

### Шаг 1: Создание проекта
1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
2. Войдите в свой Google-аккаунт.
3. В верхней панели нажмите на выпадающее меню выбора проектов (обычно написано **"Select a project"**).
4. В открывшемся окне нажмите **"New Project"**.
5. Введите имя проекта (например, `My-gAcss-Audit`) и нажмите **"Create"**. Подождите несколько секунд до завершения создания.

### Шаг 2: Включение Google Drive API
1. Убедитесь, что в верхней панели выбран именно ваш новый проект.
2. В левом меню выберите **"APIs & Services"** > **"Library"**.
3. В строке поиска введите **"Google Drive API"**.
4. Нажмите на найденный результат и нажмите кнопку **"Enable"** (Включить).

### Шаг 3: Настройка экрана согласия (Consent Screen)
1. Перейдите во вкладку **"APIs & Services"** > **"OAuth consent screen"**.
2. Выберите тип пользователя **"External"** и нажмите **"Create"**.
3. Заполните обязательные поля:
    * **App name:** `gAcss`
    * **User support email:** ваша почта.
    * **Developer contact information:** снова ваша почта в самом низу.
4. Нажимайте **"Save and Continue"** на всех этапах (Scopes, Test Users), пока не вернетесь к дашборду.

### Шаг 4: Добавление тестового пользователя
*Пока ваше приложение не прошло проверку Google, им можете пользоваться только вы.*

1. На странице **"OAuth consent screen"** найдите раздел **"Test users"**.
2. Нажмите **"Add Users"** и введите свой адрес почты Google.
3. Нажмите **"Save"**.

### Шаг 5: Создание и скачивание ключа
1. В левом меню выберите раздел **"Credentials"**.
2. Нажмите сверху **"+ Create Credentials"** > **"OAuth client ID"**.
3. В выпадающем списке **"Application type"** выберите **"Desktop App"**.
4. Введите имя (например, `gAcss Key`) и нажмите **"Create"**.
5. В появившемся окне нажмите **"Download JSON"** (иконка скачивания справа).

### Шаг 6: Настройка программы
1. Скачанный файл будет иметь длинное имя. **Переименуйте** его в `credentials.json`.
2. Создайте в папке с программой подпапку `data` и положите файл туда.

---

### ⚠️ Важное замечание при первой авторизации:
При первом запуске Google может показать предупреждение: **"Google hasn’t verified this app"**.
Это происходит потому, что вы используете свой личный ключ. Чтобы продолжить:
1. Нажмите **"Advanced"** (Дополнительно).
2. Нажмите **"Go to gAcss (unsafe)"** (Перейти к gAcss).
3. Нажмите **"Continue"** (Продолжить).

Никогда не передавайте ваш файл credentials.json посторонним лицам, так как он является ключом к данным вашего облачного хранилища.
